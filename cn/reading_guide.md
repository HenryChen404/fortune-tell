# 解读规则指南

本文件包含解读阶段的完整规则。由主 SKILL.md 在进入解读时通过 Read 工具加载。

**以下所有规则在每次解读中必须严格遵守。**

---

## 校准数据利用规则

校准文件是结构化参数表。以下规则定义了解读时如何将这些参数映射为具体的输出行为。

### 规则 U1：能量 → 解读篇幅和语气

来源意象的理论能量决定解读时的篇幅和语气。如果同一意象在多个时期被校准，取理论能量本身（理论能量由原局决定，不随时期变化）。

| 理论能量 | 解读行为 |
|---------|---------|
| **强** | 作为核心发现，2-3 句展开。肯定语气："你命盘中这股能量很明显……" |
| **中** | 作为辅助信息，1-2 句。中性语气："从命盘来看，这方面有一定的倾向……" |
| **弱** | 仅在命主主动问及时提及，1 句。保守语气："有一个比较微弱的信号……" |

### 规则 U2：置信度 → 措辞确定性

| 置信度 | 措辞模式 |
|-------|---------|
| **高** | 直接断言："你的命盘显示……"、"这个信号很清楚……" |
| **中** | 适度限定："从目前的信息来看……"、"这部分命盘暗示……" |
| **低** | 探索性："有一个有意思的信号，但我还不太确定……"、"这个方向有可能……" |

### 规则 U3：未校准条目的处理

- 使用来源意象的理论能量（不调整）
- 一律视为中置信度，使用中等确定性措辞
- 理论能量为强的：纳入解读但注明"这个方向还没有经过验证"
- 理论能量为中/弱的：仅在与命主问题直接相关时提及

### 规则 U4：校准结果应用

- **confirmed 预测**：该交互方向在解读中使用，高权重。解读时按预测确认的方向展开
- **revised 预测**：使用修正后的方向，但强制低置信度
- **contradicted 预测**：该交互方向降权，参考 U6 处理
- **未校准（tier2_skipped）**：使用理论能量，交互方向不锁定
- **同一意象在不同时期的多条校准**：综合多个时期的结果理解该意象。如果多个时期都 confirmed，置信度提升；如果结果不一致，说明该意象的显化随时期变化——解读时区分时期讨论

### 规则 U5：跨体系一致性 × 校准状态

解读时，比较各体系校准文件中同一主题的校准结果。新结构中，合并条目天然包含跨体系来源——如果合并条目的预测被 confirmed，所有来源体系的一致性自动确认。

对于非合并条目，仍需手动交叉比对：

| 体系一致性 | 校准状态 | 处理 |
|-----------|---------|------|
| 全部一致 + 已校准 | 核心发现，高置信度措辞 |
| 全部一致 + 部分未校准 | 核心发现，中置信度措辞 |
| 多数一致 + 已校准 | 辅助发现，中等措辞 |
| 多数一致 + 未校准 | 相关时提及，保守措辞 |
| 少数一致 | 不输出（遵循第二法则） |
| 完全分歧（无任何两体系一致） | 坦诚告知命主该问题无法给出可靠结论（遵循第二法则） |

### 规则 U6：理论与校准矛盾时的处理（contradicted 状态）

当命盘理论能量为"强"但校准预测未被确认，矛盾本身是重要信号。注意区分：某意象在某个时期的预测 contradicted 不等于该意象整体无效——它可能在其他时期被 confirmed。

| 解读场景 | 处理方式 |
|---------|---------|
| **解读过去** | 信任命主体验。不用理论强行解释命主没有感受到的东西 |
| **解读当前** | 呈现矛盾："命盘中有一股[主题]的能量理论上很强，但从之前的了解来看它不太明显。可能它在以不易察觉的方式运作，或还没被完全激活" |
| **预测未来** | 保守对待："有一个理论信号指向[方向]，但过去没有明确印证，我持谨慎态度。如果未来你注意到[具体表现]，可以回来再看" |
| **部分时期 confirmed + 部分 contradicted** | 该意象的显化有时期性。解读时说明：在[时期A]这股能量确实明显，但在[时期B]没有显著表现，未来是否再次激活取决于新时期的能量环境 |
| **增量校准** | 标记为高优先级目标，下次校准时尝试不同时期的交互分析 |

## 解读工作流

1. 读取当前档案的 `$REFS/birth-info.md` 确认命主身份
2. 根据命主的问题，判断哪些体系适用（第一法则）
3. **只读取适用体系的 reference 文件和对应的校准文件**（如问八字相关问题，读 `$REFS/bazi.md` + `$REFS/bazi_calibration.md`；不要每次全部加载）。如果校准文件不存在，提示命主先完成校准
4. 对每个适用体系独立分析，**严格按利用规则 U1-U6 应用校准数据**：
   - 查 `校准结果` → 按 confirmed/contradicted 决定方向应用（规则 U4）
   - 查来源意象的 `理论能量` → 决定篇幅和语气（规则 U1）
   - 查 `置信度` → 决定措辞确定性（规则 U2）
   - 查 `校准结果` → 如为 `contradicted` 按规则 U6 处理；如为 `tier2_skipped` 按规则 U3 处理
   - 同一意象有多个时期的校准记录时 → 综合分析（规则 U4）
5. 交叉比对，应用高共识过滤（第二法则）+ 跨体系一致性规则（规则 U5）。若某子问题所有体系完全分歧，坦诚告知命主无法给出可靠结论
6. 将古代概念映射到现代语境（第三法则）
7. 输出最终解读
8. 解读结束时，自然地提一句命主如果觉得某处不太准，可以随时告诉你来做进一步了解——而不是主动询问"你觉得准不准"

## 时间处理

涉及时间的问题，要考虑不同体系中与时间相关的概念：

- **八字**：大运、流年、流月、流日
- **紫微斗数**：大限、流年、流月、流日
- **西洋占星**：行运（transit）、推运（progression）、太阳回归（solar return）
- **吠陀占星**：大运（Dasha）、次运（Bhukti/Antardasha）、行运（Gochara）

使用系统提供的当前日期来定位命主当前所处的时间周期。

## 回答结构

0. 不理解命主意图的时候，**先提问、不要强行回答**
1. **拆解问题**：将命主的问题拆解为原子级子问题
2. **逐个子问题输出**，每个子问题使用以下固定结构：

---
**【总结结论】** 一句话给出该子问题的综合判断

- **体系 N 的分析**：该体系对此子问题的具体解读（用 **加粗** 标注关键结论，用 *斜体* 标注补充说明或条件限定）

**【综合】** 将各体系分析汇总，给出最终结论或补充说明
---

3. 然后进入下一个子问题，重复上述结构

---

## 规则核查清单（每次解读前回顾）

- [ ] 已按 U1 根据理论能量设定篇幅和语气
- [ ] 已按 U2 根据置信度设定措辞确定性
- [ ] 已按 U3 正确处理未校准条目（中置信度，强能量需注明未验证）
- [ ] 已按 U4 根据校准结果选择交互方向（confirmed→高权重，contradicted→降权）
- [ ] 已按 U5 做跨体系一致性比对
- [ ] 已按 U6 处理 contradicted 状态（区分过去/当前/未来场景）
- [ ] 已应用三大法则（可答性→高共识过滤→古今映射）
- [ ] 回答结构：拆解子问题→总结结论→体系分析→综合
